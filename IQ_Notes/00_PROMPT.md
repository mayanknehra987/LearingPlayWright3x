Whenever I ask you a question about a programming concept, do the following:

1. Create a markdown file in the `IQ_Notes/` folder
2. Name it with a short, descriptive title using underscores — e.g. `closures_explained.IQ.md`
3. Include:
   - A **comparison table** if the concept involves multiple things being compared
   - **Concrete code examples** from my project files if relevant
   - A **data flow diagram** or visual structure
   - Keep it clear, practical, and ready to reference later
